# Product Requirements Document (PRD)
**프로젝트명**: ImageSearch (키다리스튜디오 안드로이드 대규모 채용 기술 과제)

## 1. 개요 (Overview)
사용자가 네이버 이미지 검색 API를 통해 '만화' 관련 이미지를 탐색하고, 뷰어를 통해 확인하며 북마크할 수 있는 콘텐츠 뷰어 앱입니다. 키다리스튜디오/레진엔터테인먼트의 웹툰 서비스 특성(대용량 이미지 렌더링, 오프라인 동작, 뷰어 인터랙션)을 기술적으로 검증하는 목적을 가집니다.

## 2. 핵심 기능 목록 (Key Features)

### 2.1 메인 화면 (Main & Bookmark Tab)
*   **상단바(Top App Bar)**: 타이틀 노출, 우측 검색 아이콘 배치 (클릭 시 검색 페이지 이동)
*   **하단 탭(Bottom Navigation)**: '메인(홈)' 탭과 '북마크' 탭으로 구성
*   **[메인 탭] 컨텐츠 페이지**:
    *   네이버 이미지 검색 API (query='만화', display=50) 데이터 로드
    *   **무한 스크롤(Infinite Scroll)**: 하단 도달 시 50개씩 자동 추가 로드 (Paging)
    *   **당겨서 새로고침(Pull-to-Refresh)**: 데이터 초기화 후 재조회
    *   **북마크 토글**: 단일 또는 다중 선택 가능 (롱클릭, 체크박스 등 UX 자유 구성)
    *   **아이템 UI**: 썸네일 이미지 및 Title 텍스트 노출
*   **[북마크 탭] 컨텐츠 페이지**:
    *   메인 탭에서 북마크한 아이템 목록 노출
    *   단일 또는 다수 아이템 삭제 기능 (UX 자유 구성)

### 2.2 검색 화면 (Search Page)
*   **진입점**: 메인 화면 상단바의 검색 아이콘 클릭
*   **실시간 검색(Debounce)**: 1자 이상 입력 시, 1초 이상 추가 입력이 없을 때 자동 검색 실행
*   **검색 대상**: API 재호출이 아닌, **"메인 탭에 이미 로드된 데이터의 Title"**만을 대상으로 필터링 (로컬 검색)
*   **상태 처리**: 
    *   검색 결과가 없거나 에러 발생 시 사용자에게 명확한 피드백(Empty View / Error 시각화) 제공
    *   제공된 첨부 스크린샷(`KSTDxLEZHIN_개발자대규모채용_안드로이드(이미지).png`)의 UI 레이아웃 컨셉을 준수

### 2.3 콘텐츠 뷰어 화면 (Viewer Page)
*   **세로 스크롤 방식 (Vertical Scroll)**: 전형적인 웹툰 스크롤 뷰 구현
*   **콘텐츠 배치 구조**:
    *   클릭한 이미지의 원본 link가 Index 0(최상단)에 노출됨
    *   그 아래로 30개의 이미지가 무작위(Random)로 노출됨
*   **인터랙션(Interaction)**:
    *   이미지 확대 및 축소 (Pinch Zoom)
    *   확대 상태에서 상하좌우 드래그 (Pan)

## 3. 비기능적 요구사항 (Non-Functional Reqs)
*   **지원 환경**: Android 10 (Min SDK 29) ~ Android 14 (Target SDK 34)
*   **예외 처리**: API 호출 실패 시 명확한 피드백 (Sealed Class / Result 패턴 활용 필수)
*   **구동 환경**: 로컬 DB/Cache 기반의 오프라인 대응 (선택/가산점)
*   **기기 파편화 대응**: 태블릿/폴더블 화면 비율 대응, 기기 회전(Configuration Change) 상태 보존, 다크모드 대응
*   **성능 목표**: 다수 이미지 스크롤 및 뷰어 로드 시 Out Of Memory(OOM) 및 Jank 현상 방지 전략 구현
